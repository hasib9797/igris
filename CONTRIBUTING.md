# Contributing

## Development

1. Fork or clone the repository.
2. Create a branch for your change.
3. Keep changes focused on real, testable behavior.
4. Run backend tests and a frontend production build before opening a PR.

## Standards

- Do not add fake pages or placeholder routes.
- Prefer removing unfinished surfaces over shipping misleading UI.
- Keep privileged operations explicit and audited.
- Update the README when installation or runtime behavior changes.
